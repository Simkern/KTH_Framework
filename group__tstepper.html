Detailed Description

Time stepper for Nek5000.

Warning
This module is not upgraded to stable Nek5000 relese yet

This module implements time stepper method for linear solver in Nek5000 using userchk interface. It allows to perform number of constant lenght cycles providing the result vector to the submodule. There are two possible submodules: power iteration and spectra calculation using Arnoldi algorithm implemented in PARPACK. The initial condition for the next stepping cycle is provided by submodule. Three stepper modes are supported: direct, adjoint, direct+adjoint.

Module interface usage:
subroutine userchk
include 'TSTEP'
if (istep.eq.0) then
! initialise runtime parameters
call tstpr_param_get
endif
! main time stepper interface
call tstepper_main
return
end
Module parameters:
Global parameter list:
Parameters provided by time stepper module (include file - TIME_STEPPERD):
  • TSTMODE: (integer)
    • 0 : no time stepper
    • 1 : direct mode
    • 2 : adjoint mode
    • 3 : direct+adjoint mode (initial optimal condition)
  • TSTSTEP : (integer) frequency of calling stepper_vsolve (number of time steps)
  • TSTCMAX : (integer) maximal number of stepper cycles (meaning of this parameter depends on the applied package; arnoldi vs power iterations)
  • TSTTOL : (real) convergence tolerance (e.g. PARPACK tolerance for Ritz values)
  • TSTIFUZAWA : (logical) first step run with UZAWA
Global parameter dependency:
Parameters required by time stepper module:
  1. CONHT (for conjugated heat transfer):
    • CHCST_SC, CHCFF_V, CHCFF_T : velocity and temperature scaling factors
    • CHGRX, CHGRY, CHGRZ : gravitational acceleration
    • CHRA, CHPR : Rayleight and Prandtl numbers
  2. Nek5000 PARAM array
    • PARAM(12) - time step dt
  3. Nek5000 number of include files;
